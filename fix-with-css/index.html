Fix the Web with CSS!

  1. Safety: Removing the Horizontal Scroll

    • Website: https://www.papersource.com/
    • CSS code: .page-wrapper { overflow: hidden; }
    • Safety: Removing the horizontal bar prevents errors and reduces the chances of a slip in execution. The horizontal bar was not adding anything functionally to the website and it makes users confused on why it is even there. By using the trackpad to try and scroll left and right, it can accidentally activate the back button. This unintentionally takes the user back to the previous page, which can make them lose their place or progress.

    3.
    • Website: https://www.papersource.com/
    • CSS code: a:hover { transform: scale(0.98); }
    • Learnability: The transformation of the box when hovered over acts as a signifier that the box is clickable. By using this feature, I increased interaction and discoverability so that the user knows the box is clickable the first time. The user would initially only think to click on the SHOP NOW link in the box, but now the SHOP NOW link could be removed from the box.

    4.

    5.
